Analyzing Unicorn Companies

Tech Stack: PostgreSQL(CTEs, JOINs, Aggregations)

This SQL analysis identifies the top 3 industries that produced the most unicorn startups between 2019 and 2021. The project uses multiple CTEs to filter and summarize data from a startup funding database, joining company, industry, and valuation tables.

Key Insights:

  • The top 3 industries with the most unicorns between 2019 and 2021 were consistently dominant in both count and valuation.
  • 2021 saw the highest spike in average unicorn valuation, suggesting a funding surge during the pandemic rebound.
  • Some industries had fewer unicorns but significantly higher average valuations, pointing to high investor confidence in specific sectors.

AI Integrated Inventory Management System (AI-IIMS)

Tech Stack: Python (Pandas, Prophet, FastAPI), Next.js, React.js, Tailwind CSS, Git/GitHub

AI-IIMS is a full-stack inventory forecasting solution developed in Agile sprints. The system forecasts demand for ingredients by analyzing historical sales data at the menu item level and translating those forecasts into ingredient quantities based on each recipe.

Deliverables:

  • Data ingestion and cleaning pipeline using Python and OpenPyXL
  • Forecasting engine using Prophet to predict daily demand for menu items and required ingredients
  • Recipe-to-ingredient mapping logic to calculate daily ingredient usage
  • RESTful API with FastAPI to handle user queries and serve predictions to the frontend
  • Interactive frontend using React and Tailwind CSS for input, selection, and visualization
  • Full project documentation such as; project scope, team charter, personas, user stories, meeting minutes, and final presentation

Analyzing Yelp Reviews

Tech Stack: Python (Pandas, Scikit-learn, TfidfVectorizer, Naive Bayes), Seaborn, Matplotlib

This project applies natural language processing (NLP) to predict Yelp review ratings based on the text content. It uses TF-IDF for feature extraction and a Naive Bayes classifier for multi-class classification. Additional analysis includes user engagement based on “cool,” “useful,” and “funny” votes.

Key Insights:

  • The model performed best on 4-star and 5-star reviews, with limited success in predicting lower ratings.
  • Skewed distribution of reviews and class imbalance affected performance, with many reviews clustering at the higher end.
  • Text content alone may not be sufficient for fine-grained rating prediction; combining other features like business categories or review length could improve results.

Deliverables:

  • Exploratory data analysis of Yelp reviews, rating distributions, and vote metrics
  • Preprocessed review text (lowercasing, stop word removal, TF-IDF transformation)
  • Naive Bayes classification pipeline trained on 5-star rating prediction
  • Model evaluation with confusion matrix and classification report with Weighted F1-score of 0.36

Ask TarotGPT

Tech Stack: ChatGPT API, Prompt Engineering, JavaScript (Prototype)

This project explores the use of GPT models to simulate tarot readings through refined prompt engineering. Ask TarotGPT generates structured 1-card, 3-card, and 5-card spreads tailored to user intentions, such as career, love, or self-reflection. The current version is prompt-based and optimized for consistency, tone, and symbolic interpretation.

Future Improvements:

  • Build a fully interactive web interface for card selection and spread visualization
  • Integrate session memory to personalize multi-question readings
  • Incorporate probabilistic spread generation using pseudo-random logic
  • Allow users to journal and revisit past readings via local browser storage
